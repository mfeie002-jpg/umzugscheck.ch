## Part 1: Philosophy — The AI-Designed Relocation System

### Why Moving Is Broken

Humans relocate the way they do because we collectively decided that stress + spreadsheets + "Umzugsfirma Zürich" Google searches should be a rite of passage. It's not the best way—it's just the way that survived.

A move isn't "finding a mover." It's a **multi-party, high-risk project** with three real jobs-to-be-done:

1. **Reduce uncertainty** (price surprises, damage, timing failures, admin mistakes)
2. **Reduce cognitive load** (humans hate decisions when stressed)
3. **Coordinate execution** (scheduling, access, permits, handover, payments)

### The "Google → Form → Phone Call" Model Is Dead

Traditional lead funnels optimize for **lead volume**, not **outcome quality**. They create:
- Price anxiety (no fixed quotes)
- Decision paralysis (compare 7 PDFs?)
- Trust gaps (escrow? insurance? who's liable?)
- Admin burden (you do everything yourself)

### The Relo-OS Paradigm

The best system is a **Relocation Operating System**: one place that orchestrates everything end-to-end.

**Golden Rule:** The best UX is not "form vs chatbot"—it's **structured automation + human rescue**.

- Chat for intent + reassurance + ambiguity
- Micro-forms for exact inputs (date, addresses, floors)
- Camera/LiDAR scan for inventory
- Human concierge only when it matters (high value, edge cases, senior dignity)

## Part 2: Technical Architecture — The 6-Phase Journey

### Phase 1: ROUTE (Intent Capture)
**Goal:** Start the process in under 30 seconds.

**Inputs (minimum viable truth):**
- From/To (address or area)
- Date window (not a single date)
- Size signal (rooms OR quick scan)
- Special constraints (lift? piano? fragile?)

**UI:** 1 screen, 4 answers, done.

### Phase 2: INVENTORY (AI Video Scan + Digital Twin)
**Goal:** Capture inventory without typing.

**Methods:**
- Smartphone video walkthrough (works for everyone)
- LiDAR-enhanced scan (iPhone/iPad Pro with RoomPlan API)

**Outputs:**
- Volume estimate (m³)
- Item categories + special items
- Access constraints
- Packing complexity score

### Phase 3: QUOTE (Instant Fixed-Price)
**Goal:** Credible fixed price, fast.

**Progression:**
1. Show price range first (anchoring, reduces fear)
2. Fixed price when inventory confidence is high
3. Fallback: "Fixed price after 2-minute video scan"

### Phase 4: BOOKING (Quality-Weighted Bidding + Escrow)
**Goal:** Single-tap booking, not a beauty contest.

**Recommendations:**
- 1 default ("Best overall")
- 2 alternatives: Cheapest acceptable, Premium

**Under the hood:**
- Quality-weighted bidding algorithm
- Smart Escrow / Treuhand integration

### Phase 5: MOVING (Live GPS Tracking)
**Goal:** Trust through transparency.

**Features:**
- Live timeline: Team → Loading → En Route → Unloading
- ETA updates
- Exception handling with priced change orders

### Phase 6: COMPLETE (Swiss Handover Protocol)
**Goal:** Perfect finish, zero admin.

**Features:**
- Digital handover checklist with photos/signatures
- Automatic escrow release on confirmation
- Swiss Admin Autopilot:
  - eUmzugCH integration (eCH-0221)
  - Swiss Post address forwarding (T-7 reminder)
  - Serafe notification
